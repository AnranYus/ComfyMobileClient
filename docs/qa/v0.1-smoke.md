## 0. 适用范围

- 目标产物：CI workflow_dispatch 出的 `androidApp-debug.apk`，`versionCode=2 versionName="0.1.0"`。
- 设备矩阵（详见 §10）：
  - **主 gate**：@Nothing 真机（OPLUS，Android 12+ ROM，已抓到 P0 闪退日志，msg `648aff8e`）。
  - **minSdk gate**：本机 ARM TCG emulator，API 24（Android 7.0 — 与 `gradle/libs.versions.toml: android-minSdk = 24` 对齐）。
  - **A14 缺口**：v0.1 不强制 API 34 真机覆盖。`compileSdk = targetSdk = 34`，OPLUS 真机 ROM 已是 12+，static path 已覆盖；attach APK 时显式标注此缺口。
- 不在范围：
  - Phase 3 节点拓扑编辑（v0.1 不解锁）。
  - Play Store 签名 / 上架（v0.3+）。
  - iOS xcarchive（v0.2 macOS CI 跑）。
- 复用规则：每个后续 v0.x release，先 fork 本文件改 §1 包元数据 + §10 设备矩阵即可。

| ID | 测试方法 | 期望 |

## 10. 设备/模拟器矩阵

| 档位 | 设备 | 谁跑 | 覆盖 |
| --- | --- | --- | --- |
| **主 gate** | @Nothing OPLUS 真机（Android 12+） | @Nothing 装机 + 跑 §2 V-11 ~ V-15（必跑），其余按手感跑 | P0 真复现 / 真机 main-thread enforcement / 实际 ROM 行为 |
| **minSdk gate** | 本机 emulator（API 24 ARM v7a TCG） | @Longly | minSdk 路径 / `mipmap-anydpi-v26` adaptive icon fallback / 旧 `Lifecycle` / 旧 `WindowInsets` / 旧 storage API |
| **A14 缺口** | — | — | v0.1 显式不覆盖；attach APK 时在 #ComfyMobile 主帖注明 |

模拟器准备命令（@Longly 本机）：

```bash
export ANDROID_SDK_ROOT=/home/anran/android-sdk
export PATH=$ANDROID_SDK_ROOT/cmdline-tools/latest/bin:$ANDROID_SDK_ROOT/platform-tools:$ANDROID_SDK_ROOT/emulator:$PATH
avdmanager create avd -n v01-api24-arm -k "system-images;android-24;default;armeabi-v7a" --device "pixel"
emulator -avd v01-api24-arm -no-window -no-audio -no-snapshot -gpu swiftshader_indirect
```

注：本机为 QEMU VM，无 KVM，emulator 走纯 TCG 翻译，冷启动可能 10~15 分钟，跑 MVP loop 慢 10x。失败回退方案：仅做 `adb install -r` + V-11 / V-12 单帧 logcat 校验，不强跑全 loop。

## 11. 异常采集模板

任何步骤失败时，往 #ComfyMobile 主帖发以下三件套：

1. **失败的 V-id** + 设备 + 重现步骤一句话
2. **logcat**：
   ```bash
   adb logcat -d -v threadtime AndroidRuntime:E ComfyMobile:V *:S | tail -200
   ```
3. **截图**（如果是 UI 问题）

无 crash 时无需贴 logcat。

## 12. 主要风险（v0.1 已知）

- **R-01**：A14 未覆盖 — 任何 API 31~34 新引入的限制（前台服务、partial photo picker、predictive back、scoped storage 收紧等）可能 v0.1 漏跑。@Nothing 如手边有 A14 真机可抽测。
- **R-02**：OPLUS ROM 上 `OplusPredictiveBackController: NoSuchMethodException` 在 P0 logcat 中已出现 — 非致命，但 ROM 私有 hook 可能在其他屏触发新栈，关注 logcat 中任何 `Oplus` 前缀的 `E` 级日志。
- **R-03**：本机 emulator 软模拟极慢，minSdk gate 实际可能只能完成"装上 + 启动 + Library"前几步，run-loop 不一定能跑完。文档中已说明可降级。
- **R-04**：debug-signed APK 装机时部分 ROM 会触发"未知来源 / 安全风险"二次确认，非 bug，@Nothing 装机时手动放行。
