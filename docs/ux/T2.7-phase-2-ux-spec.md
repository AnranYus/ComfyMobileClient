## 1. Graph rendering visual spec (#23 / #24)

### 1.1 Node shape

- **Container**: rounded rectangle, corner radius 12dp, elevation 2dp, surface color `surface` with 1dp `outline` border.
- **Width**: 200dp default. Nodes with no editable params (e.g. `VAEDecode`) collapse to 160dp.
- **Height**: variable; minimum 56dp (title bar only), grows to fit summary rows.
- **Selected state**: border 2dp `primary`, elevation 6dp.
- **Whitelist node** (in v1.json): full body shown.
- **Unknown node**: collapsed body (title bar only) with `surfaceVariant` background, italic title to signal "not editable on mobile". Ports still drawn so connections render correctly.

### 1.2 Title bar

- 40dp tall, padding 12dp horizontal.
- Left: classType displayName (resolved from `NodeDescriptorRegistry.lookup(classType)?.displayName` or fallback to raw `classType` string for unknown nodes).
- Right: status icon (single icon, sized 20dp):
  | State | Icon | Color role |
  |---|---|---|
  | idle | (none) | — |
  | currently running | rotating spinner | `primary` |
  | completed in this run | ✓ | `tertiary` |
  | cached (from `execution_cached`) | ⚡ | `tertiaryContainer` (Material 3 low-saturation tertiary — "quietly completed") |
  | errored | ⚠ | `error` |

### 1.3 Body — summary rows (read-only)

For whitelist nodes only. Renders the **summary view** of the editable params on the graph itself, so the user can scan a workflow's settings without opening the drawer.

- One row per `editableParam`, height 28dp, padding 12dp horizontal.
- Layout: `[label]            [compact value preview]` (label left, value right-aligned).
- Compact value rendering:
  - `Slider` / `Number` / `Integer`: value as plain text + unit if any (`30` for steps, `1.0` for cfg).
  - `Toggle`: `开` / `关` (zh) / `On` / `Off` (en).
  - `SingleLineText` / `MultilineText`: first 24 chars + `…` if longer; em-dash if empty.
  - `Dropdown`: selected value (resolved via `source` if necessary, else raw).
  - `ModelPicker`: model filename (basename without extension).
  - `ImagePicker`: thumbnail 20dp square if loaded, else `🖼`.
  - `Hidden`: not rendered in summary (but still preserved in workflow JSON).

If body has more than 4 summary rows, show first 3 and a `…N more` row to keep node compact.

### 1.4 Ports (sockets)

- 12dp circle, 6dp from node edge.
- Inputs on left edge, outputs on right edge, vertically distributed.
- Color-coded by ComfyUI link type (matches desktop ComfyUI defaults):

| Type | Color role |
|---|---|
| MODEL | purple |
| CLIP | blue |
| VAE | red |
| LATENT | green |
| IMAGE | orange |
| MASK | dark green |
| CONDITIONING | beige |
| CONTROL_NET | dark purple |
| (unknown / other) | `outline` gray |

Port label: not shown by default; tap-and-hold a port shows a tooltip with the link type name (a11y).

### 1.5 Connections (links)

- Cubic Bezier curve from output port (right side of source node) to input port (left side of target node).
- Stroke: 2dp; color = source port type color.
- **LOD degradation contract** (per ADR-0002 step 3, formalized in T2.1b): during any active interactive gesture in Phase 2 — pan / zoom / node-drag — connection layer downgrades from Bezier to **straight line**; restore to Bezier on gesture release. T2.1a ships steady-state Bezier only; T2.1b owns the degradation toggle. §1.8 below restates this constraint as a perf gate.
- Selected node's incident edges: stroke 3dp + `primary` accent overlay.

### 1.6 Node positioning

- Read `pos` field from each node in the imported `WorkflowEnvelope.original` (UI format).
- API-format-only imports (no `pos`): apply auto-layout — left-to-right BFS from nodes with no inputs, vertical stagger by depth. Phase 2 doesn't require fancy auto-layout; "good enough that the user can read it" is fine.

### 1.7 Canvas controls (overlay, top-right)

- **Reset view** button: re-centers and rescales to fit all nodes within viewport.
- **Fit-to-selection** button (only when a node is selected): re-frames around the selected node + 1-degree neighbors.
- Buttons are 48dp circular, `surface` with elevation, fade out 2s after canvas idle (non-aggressive).

### 1.8 Performance constraints (gate per T0.5)

- Re-state `docs/ux/T0.3-ux-mobile-comfy.md` §2 hard requirements:
  - 30 nodes visible @ 60fps drag on Pixel 6a / iPhone 12.
  - Viewport virtualization (only draw nodes whose bbox intersects expanded viewport + 1 screen of buffer).
  - Static node body cached as `ImageBitmap`; redraw on state change only, not every pan tick.
  - Connection-layer downgrade during **any active interactive gesture (pan / zoom / node-drag)**: Bezier → straight line; restore to Bezier on gesture release. (Matches the LOD contract stated in §1.5; T2.1b owns the implementation.)

### 1.9 First-frame onboarding hint

When a user opens a workflow for the first time after import, overlay a one-shot tooltip layer:
- "捏合缩放、双指拖动平移；轻点节点选中，长按查看参数。"
- "Pinch to zoom, two-finger pan. Tap to select, long-press to see params."

Dismissible by tap; suppressed for subsequent workflows in the session.

## 2. Param editor drawer (#25)

### 2.1 Surface and trigger

- **Phone**: bottom sheet, 60% screen height by default; user can drag handle to expand to 90% or collapse to 30%.
- **Tablet**: right-side panel, 360dp wide, persistent while a node is selected.
- **Trigger**: long-press a whitelist node, OR tap an already-selected whitelist node.
- **Non-whitelist nodes**: trigger toast `此节点暂不支持移动端编辑，可在桌面 ComfyUI 中调整后重新导入` / `This node can't be edited on mobile. Adjust it in desktop ComfyUI and re-import.` (per T0.3 Q1 safety valve), don't open drawer.

### 2.2 Drawer layout

```
┌───── (drag handle, bottom sheet only) ─────┐
│  [classType displayName]                ✕  │   ← header, 56dp, title left, dismiss right
├────────────────────────────────────────────┤
│  [param 1 label]                           │   ← rows scroll vertically
│  [param 1 control]                         │
│  [param 1 help text (collapsed by default)]│
│  ───────                                   │
│  [param 2 label]                           │
│  [param 2 control]                         │
│  [param 2 help text]                       │
│  ...                                       │
└────────────────────────────────────────────┘
```

- Header dismiss: ✕ icon → close drawer; tap outside drawer dismisses too.
- Help text: gray small text below each control; hidden behind a `?` icon by default; tap to expand inline.

### 2.3 Param row layout

- **Label**: `param.displayName` resolved by locale, body-medium, bold-medium weight. 4dp bottom margin.
- **Control**: per the catalog in `T0.4-risk-ux-and-connection-variants.md` §2.1.
- **Help**: `param.helpText` resolved by locale. Default collapsed; expanded state remembered for the session.
- **Order**: declaration order in `v1.json`. KSampler order = `seed, steps, cfg, sampler_name, scheduler, denoise` per the descriptor.

### 2.4 Control-specific Phase 2 details

| Control | Phase 2 detail |
|---|---|
| **Slider** | If `presets` present, render preset chips below the slider track. Tap chip → snap value + brief haptic. Preset values clamped to `[min, max]`. |
| **Slider with EmptyLatentImage width/height** | Pair the two sliders horizontally with a single "lock aspect ratio" toggle to the right (post-MVP soft-add; Phase 2 may skip and just show two independent sliders). |
| **Integer (seed)** | Render with the **Seed special** UI variant: number field + 🎲 random + 🔒 lock. Lock=on disables 🎲 and prevents auto-seed-on-rerun. |
| **Integer (batch_size etc.)** | Stepper variant: −/+ buttons either side of the number field. |
| **MultilineText (prompt)** | Auto-expanding height, min 80dp, max 50% screen. Supports `embedding:` and `<lora:` triggers per `descriptor.autocomplete`. Triggers open a bottom-sheet picker (model name list); commit inserts `embedding:NAME` or `<lora:NAME:1.0>` at cursor. |
| **Dropdown** | Show `加载中...` / `Loading…` while `source` data is fetching from `/object_info`. On fetch fail, show inline retry button + `选项加载失败` / `Couldn't load options`. |
| **ModelPicker** | Open a full-screen picker: search bar at top + grouped list (`最近使用` / `Recent`, `全部` / `All`, `收藏` / `Favorites`). Each entry shows model filename + file size if available. If the workflow's selected model is **missing on server**, the trigger shows a warning chip `未安装` / `Not installed` + opens picker on tap. |
| **ImagePicker** | Trigger shows current thumbnail (60dp square) + filename. Tap opens action sheet: `相册` / `Gallery`, `相机` / `Camera`, `粘贴` / `Paste`, `重置` / `Reset`. Selected image uploads to ComfyUI via `/upload/image` and the param is set to the returned filename. |

### 2.5 Apply / dirty state

- **No commit until Apply**: edits are local to the drawer until user taps `应用` / `Apply` in the drawer footer (sticky at bottom). This lets users abandon unintended changes by closing the drawer.
- Footer: `[取消 / Cancel]   [应用 / Apply]`. Cancel → dismiss without writing. Apply → patch `WorkflowEnvelope` and dismiss.
- Visual dirty indicator: when at least one control has been edited, `应用 / Apply` button is `primary` filled; otherwise it's `outlined` and disabled.
- Dismissing without Apply (swipe down / tap outside / ✕): if dirty, show a confirm dialog `丢弃改动？` / `Discard changes?` with `丢弃` / `Discard` and `继续编辑` / `Keep editing` actions.

### 2.6 Validation

- Per-control validation runs on edit, with 300ms debounce (mirror connect-form rule from `T1.4-connection-ui-spec.md` §3).
- Apply is disabled if any control has a validation error.
- For sliders/numbers: `min <= value <= max` enforced by the control widget; no additional validation needed.
- For prompts: no length cap in v1 (server enforces).

## 3. Run / waiting / interrupted / failed state UX (#26)

This section **builds on** `T0.4-risk-ux-and-connection-variants.md` §3 (which already has the A/B/C connection branch design) and adds Phase 2 surface specifics.

### 3.1 Surface

When the user taps the run CTA:
- **Phone**: full-screen takeover. Workflow title in app bar; cancel CTA top-right.
- **Tablet**: right-panel takeover (graph remains visible left); cancel CTA top-right of right panel.

### 3.2 Components

- **Top status row** (per T0.4 §3.2): workflow name + "运行中" / "Running" tag + step bar `current/total` + ETA tag.
- **Node graph annotation**: per T0.4 §3.2 — currently-executing node accent border + spinner; cached nodes ⚡; completed nodes ✓.
- **Bottom progress detail card** (collapsible): current node displayName + `(value/max)` from `progress` event + thumbnail of last `executed` output if available.
- **Cancel button** (sticky top-right, destructive `error` color): tap → confirm sheet `确定取消？已生成的部分将丢失。` → confirm → `POST /interrupt` (running) or `POST /queue {delete:[id]}` (queued) per T0.4 §3.2.

### 3.3 Branch B / C banners

(Per T0.4 §3.3 / §3.4 — already specced. Phase 2 adds: banner placement is **above** the top status row, not on top of the graph annotation.)

### 3.4 Result transition

- On `execution_success`: top status row collapses; transition to **§4 Output gallery**. The graph view remains accessible behind the gallery (back button returns to graph).
- On `execution_error`: collapse status row; show full-screen error sheet with title (`生成失败` / `Generation failed`), the failing node displayName + `traceback` collapsed, primary CTA `再试一次` / `Try again` (re-submits same prompt).
- On `execution_interrupted` (cancel): show `已取消` / `Cancelled` toast; return to graph.

## 4. Output gallery (#27)

### 4.1 When does it open

- After `execution_success` completes — auto-transition.
- From history (#28) — tap a history entry to open its outputs.

### 4.2 Single output

- Full-screen image viewer (pinch-to-zoom, double-tap to zoom).
- Overlay: top app bar with back arrow + workflow name; bottom action row.
- **Action row** (left to right):
  - 保存到相册 / Save to gallery (post-MVP — see §4.5)
  - 分享 / Share (system share-sheet with image — second-segment wiring, see §4.5)
  - 收藏 / Favorite (toggles flag in local job store — second-segment wiring, see §4.5)
  - 用此工作流再生 / Run again (re-submits same prompt, fresh seed)
  - 微调再生 / Tweak & run (returns to graph view, drawer pre-opens on a sensible target — e.g. `KSampler` to nudge seed/steps; user can then run again)
- **Metadata footer** (collapsible, default collapsed):
  - Model name (from `CheckpointLoaderSimple.ckpt_name`)
  - Prompt (first 200 chars + `展开` / `Expand`)
  - Seed (with copy button)
  - Dimensions
  - Time elapsed (start → success)
  - "查看完整工作流" / "View full workflow" link → returns to graph view

### 4.3 Multiple outputs (batch_size > 1)

- Grid: 2 columns on phone, 3-4 columns on tablet.
- Tap a thumbnail → open as single output (back returns to grid).
- Selection: long-press toggles select mode, allowing batch save / share / delete.
- Same metadata footer applies to the batch as a whole (one prompt, one seed range).

### 4.4 Empty state

- "首次生成会出现在这里" / "Your generations will appear here." — only relevant when entered from a history with no outputs (rare edge case).

### 4.5 Disabled-placeholder action group (Phase 2 first-segment scope)

The action row contains three buttons that ship in the **first** segment of T2.4 as **visible-but-disabled placeholders**. They share the same UX treatment: rendered as `OutlinedButton(enabled = false)` with no side effect on click. Real wiring lands in a second segment (or a post-MVP iteration), per the agreed split (Lily/Ores msg b0b2cee1, Andy PR #27 deferral).

| Button | First-segment status | Real wiring blocker |
|---|---|---|
| 保存到相册 / Save to gallery | disabled placeholder | post-MVP (photo library permission UX not in T0.5 gates, per `T0.3-ux-mobile-comfy.md` §1) |
| 分享 / Share | disabled placeholder | needs platform share-sheet `expect/actual` (Android `Intent.ACTION_SEND` / iOS `UIActivityViewController`) — second-segment wiring |
| 收藏 / Favorite | disabled placeholder | needs JobRepository schema extension (`is_favorite` flag) + history-view filter chip — second-segment wiring |

**Phase 2 first-segment contract** (already enforced by PR #27 / `OutputGalleryScreen.kt`):
- `enabled = false` so the button doesn't visually claim functionality.
- `onClick = {}` so no permission prompt, share intent, or repository mutation fires by accident.
- Tooltip `下一个版本支持` / `Coming in next version` available via `OutputGalleryCopy.comingSoon`.
- a11y: the disabled buttons are still focusable and announced as disabled — the user sees that these affordances are planned, just not active yet.

**Second-segment wiring scope** (separate PR, after T2.3 / T2.6 land or as a focused follow-up):
- Share: implement platform `expect/actual` share-sheet bridge; build a JPEG/PNG payload from the resolved image URL; wire `onShare` on the action.
- Favorite: extend SQLDelight `job` table with `is_favorite` column; expose toggle via `JobRepository.toggleFavorite(promptId)`; surface a "收藏 / Favorites" filter chip in T2.5 history.
- Save to gallery: stays **post-MVP**. Requires Android `WRITE_EXTERNAL_STORAGE` / scoped storage + iOS `PHPhotoLibrary` permission UX. Track in a Phase 4+ task; do not enable in Phase 2 second segment unless that decision is revisited.

## 5. History view (#28)

### 5.1 Surface

- Tab in main bottom navigation (phone) or sidebar entry (tablet).
- Title: `历史` / `History`.

### 5.2 List

- Reverse-chronological by `submittedAt` from `Job` table.
- Each row, 80dp tall:
  - Left: 64dp square thumbnail (Coil load via `ComfyImageMapper` from job's first output) — fallback grey placeholder if no output yet (running) or load failed.
  - Middle: workflow name (top), prompt snippet (first 60 chars + ellipsis), submission time (relative — `刚刚` / `just now` / `2 小时前` / `2h ago`).
  - Right: status icon (✓ success / ⚠ failed / ⟳ running / ✕ cancelled).
- Tap → open Output gallery for that job.
- Long-press → action menu: `重命名` / `Rename`, `打开工作流` / `Open workflow` (loads the saved workflow + params), `删除` / `Delete`.

### 5.3 Filtering

Top of list, sticky chips:
- `全部` / `All` (default)
- `收藏` / `Favorites` (rows where `Job.isFavorite == true` — corresponds to the second-segment wiring promised in §4.5; toggled from the gallery action row's Favorite button)
- `成功` / `Successful`
- `进行中` / `Running` (groups in_progress + queued)
- `失败/取消` / `Failed/Cancelled`

Workflow filter: small `按工作流` / `By workflow` dropdown if user has many entries (Phase 2 may skip, ship in subsequent iteration).

### 5.4 Empty state

`还没有生成记录。试试在工作流里点"生图"。` / `No generations yet. Try tapping "Generate" in a workflow.`

### 5.5 Ghost-state defense (per T0.5 + T1.3)

- `JobReconciler` already runs at app launch (T1.3). The history list **must** subscribe to a Flow that reflects reconciliation results, so a job that was "running" at last app close transitions to its true terminal state on next view.
- During reconciliation (first 1-2 seconds after app launch), `进行中` filter chip shows a subtle dot indicator.

## 6. Workflow import + library (#22 + #29)

### 6.1 Library screen (entry)

- Top app bar: title `工作流` / `Workflows` + connection status indicator (right side, mini).
- FAB: "+", opens import action sheet (§6.2).
- List: saved workflows from SQLDelight workflow table (T2.6 / #29).
- Each row, 72dp tall:
  - Left: 56dp square thumbnail (last successful output for this workflow, or generic `🧩` icon if never run).
  - Middle: workflow name (top), node count + last-run summary (`12 节点 · 上次 2h 前` / `12 nodes · last run 2h ago`).
  - Right: chevron.
- Tap → open graph view for that workflow.
- Long-press → action menu: `重命名` / `Rename`, `导出 JSON` / `Export JSON` (writes to file picker), `删除` / `Delete`.

### 6.2 Import action sheet

Triggered from FAB or system share-sheet receipt. Bottom sheet:
- `从文件选择 (JSON / PNG)` / `Pick file (JSON / PNG)`
- `从剪贴板粘贴 JSON` / `Paste JSON from clipboard`
- `相册中的 PNG` / `PNG from gallery` (tries to extract `extra_pnginfo.workflow`)
- `取消` / `Cancel`

System share-sheet receiving JSON / PNG: same flow as above but skip the action sheet and go straight to import preview.

### 6.3 Import preview

After source selected, show a preview screen:
- Workflow name (default = filename without extension; user can edit before save).
- Node count + breakdown (e.g. `12 个节点（10 个支持移动端编辑，2 个未知）` / `12 nodes (10 editable on mobile, 2 unknown)`).
- Server compatibility note: if any whitelist node references a model not present on the active server, warn `部分模型未安装：xxx.safetensors（导入后可在节点里替换）` / `Some models aren't installed: xxx.safetensors (you can swap them after import)`.
- Two CTAs: `导入` / `Import` (primary) and `取消` / `Cancel`.

### 6.4 Import errors

| Cause | Title (zh / en) | Body |
|---|---|---|
| File not parseable as JSON | `文件不是有效的 JSON` / `Not a valid JSON file` | `选择一个 ComfyUI 导出的 JSON 或 PNG。` / `Pick a ComfyUI-exported JSON or PNG.` |
| PNG without `extra_pnginfo.workflow` chunk | `这张 PNG 里没有工作流` / `No workflow embedded in this PNG` | `不是所有 PNG 都内嵌工作流。试试 ComfyUI 直接导出的图片。` / `Not all PNGs carry a workflow. Try a ComfyUI-exported image.` |
| Workflow has zero nodes | `工作流是空的` / `Workflow is empty` | `检查源文件，或换一个工作流试试。` / `Check the source file, or try another workflow.` |
| Unsupported version (future) | `暂不支持这个工作流版本` / `Unsupported workflow version` | `升级 App 后再试。` / `Try again after updating the app.` |

### 6.5 Conflict resolution

When importing and a workflow with the same name already exists:
- Show modal `已有同名工作流` / `Workflow name already exists`.
- Three options:
  - `替换` / `Replace` — overwrite. **Warning**: also asks "这会丢失之前的历史关联吗？" — actually no, history is by `jobId`, not workflow name; show `（不会影响历史记录）` / `(History records are kept)` reassurance.
  - `另存为新工作流` / `Save as new workflow` — auto-rename `(2)`, `(3)` etc.
  - `取消` / `Cancel`.

### 6.6 Import progress UX

For large workflows (>50 nodes, rare in casual user persona), import parsing may take a moment. Show `导入中…` / `Importing…` spinner. Most imports are sub-second; spinner only renders if parse exceeds 200ms.

## 7. Cross-surface consistency

### 7.1 Top status indicator

Always visible (top app bar right side) on all post-connect surfaces:
- Connected: `tertiary` dot (small, 8dp) + friendly server name (only on tablet; phone hides text to save space).
- Reconnecting: pulsing dot.
- Lost: `error` dot + `重连` / `Reconnect` mini button.

Tap status indicator → open server management (lets user disconnect / switch server).

### 7.2 Error banner

Top of screen, dismissable, slides down from top edge. Use for **transient** errors (failed model fetch, save failed, etc.). Permanent errors (Lost connection, generation failed) use full-screen sheets.

### 7.3 Loading state taxonomy

| Scope | Treatment |
|---|---|
| Inline (within a row / control) | small circular spinner inside the control |
| Sheet content (drawer, modal) | inline spinner top-center, content below |
| Full-screen blocking | only for: app launch, switching active server, importing large file, submitting prompt. Other operations should stay inline. |

### 7.4 Empty-state taxonomy

Pattern: large neutral icon + single-sentence headline + optional secondary CTA.
- Icon: `outline`, 48dp.
- Headline: body-large, no exclamation marks.
- CTA: `outlined` button if action is available; nothing if it's just a "wait for content" state.

### 7.5 Haptics

Light haptic on:
- Long-press triggering drawer
- Slider preset chip snap
- Status transition (Connected → Reconnecting, etc.)

No haptic on routine taps (would feel noisy).

## 8. Out-of-Phase-2 scope (intentionally not designed)

These remain post-MVP per Phase 0 / Phase 1 decisions:
- mDNS auto-discovery (placeholder only, in connect screen).
- Save-to-gallery permission UX.
- Multi-server simultaneous connections.
- Editing topology of any node (whitelist or unknown) — read-only graph until v1.5+.

- Custom node descriptor extension (user-supplied descriptors) — Phase 4+.

## 9. Phase 2 acceptance contract (cross-ref T0.5 + T2.8)

DoD for the MVP loop, mirroring Priestess's Phase 2 completion criteria:
1. **Import → Run → Output → History** end-to-end on Android device.
2. **30-node workflow** scrolls/zooms ≥ 60fps on Pixel 6a / iPhone 12 (T0.5 perf gate, also covered by #31).
3. **PNG roundtrip**: export from ComfyUI desktop → import on mobile → run → exported PNG retains workflow embed (covered by T0.5 PNG fixture).
4. **iOS Simulator** path runs an equivalent loop.
5. **Whitelist param edits**: each control type renders, validates, persists, and shows up in API submission per descriptor v1.json.
