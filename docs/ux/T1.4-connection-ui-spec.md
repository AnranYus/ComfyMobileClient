## 2. ConnectionState → UI mapping

Reference Kotlin types live in `shared/src/commonMain/kotlin/com/comfymobile/data/network/ConnectionState.kt`.

| State | Indicator | Banner | User-visible copy | Dismissible | Color role |
|---|---|---|---|---|---|
| `Connected` | Green dot + friendly server name | none | (none) | n/a | `tertiary` (success) |
| `Reconnecting(LAN_FLAKE)` | Pulsing dot (1.5s cycle) on existing indicator | **silent** — no top banner | inline microcopy: `网络小波动，正在重连…` / `Brief network blip, reconnecting…` | n/a (auto-recovers) | `surfaceVariant` (subtle) |
| `Reconnecting(BACKGROUND_RESUMED)` | Pulsing dot | **explicit banner**, top of screen | banner: `欢迎回来，正在检查你的生成…` / `Welcome back, checking your generation…` | non-dismissible (auto on resolve) | `secondaryContainer` (info, warm) |
| `Lost(error)` | Red dot | `ConnectErrorView` overlay or full-screen | per `ConnectError` (§5) + retry CTA | dismissible to retry screen | `errorContainer` (alarm but not aggressive) |

**ETA tolerance during Reconnecting** (from T0.4 §3.5): freeze the last-known progress; do not zero ETA; resume smooth interpolation only after a fresh `progress` event.

**Cancel button during Reconnecting**: keep visible but `enabled=false` with tooltip `等待重连后可取消`. Re-enable on `Lost` (semantic shifts to "abandon this job").

## 3. Form validation rules

`ServerForm` fields and rules. All errors render inline (red `outlineColor=error`, helper text below field).

| Field | Validation | Inline error copy (zh / en) |
|---|---|---|
| host | non-empty after trim | `请输入服务器地址` / `Please enter a server address` |
| host | optional: warn if not IPv4 / hostname pattern | (no hard block — let backend probe decide) |
| port | integer in `[1, 65535]` | `端口需为 1–65535` / `Port must be 1–65535` |
| port | default value `8188` (placeholder, prefilled) | — |
| friendly_name | optional; if provided, length `1..30` after trim | `名字最长 30 个字符` / `Name must be 1–30 chars` |
| friendly_name | optional: warn (not block) on duplicate | inline below field: `这个名字已经用过` / `That name is already in use` (still allows submit; user choice) |

**Submit button state**:
- disabled while any required-field error active
- shows loading spinner during connect attempt (`ConnectionState.Connecting` — Alice may add intermediate state to driver if not present)
- on success → close form, navigate to next surface (post-Phase-2)

**Per-field debounce**: 300ms before showing inline error after user types (avoid flicker while typing).

## 5. ConnectError → copy + action catalog

`ConnectError` enum (in `shared/src/commonMain/kotlin/com/comfymobile/data/network/ConnectError.kt`):
`FORMAT, TIMEOUT, REFUSED, TLS_HANDSHAKE, NOT_COMFYUI, WRONG_PORT_404, NO_ACTIVE_SERVER, UNKNOWN`.

For each: a title, a body, an actionable suggestion, a primary CTA label, and a primary **action enum** (`PrimaryAction { RETRY, DISMISS }` — declared on `ConnectErrorCopy.Lookup`). The action drives the button click handler in `ConnectErrorView`: `RETRY` calls `onRetry`, `DISMISS` calls `onDismissError`. Default is `RETRY`. The error context (`ConnectErrorContext.description` and `ConnectErrorContext.statusCode`) renders as collapsed "技术细节 / Technical details" below the body.

| ConnectError | Title (zh / en) | Body (zh / en) | Suggested action (zh / en) | Primary CTA (label) | Action |
|---|---|---|---|---|---|
| FORMAT | `地址格式不对` / `Invalid address` | `检查 IP 和端口的格式（示例：192.168.1.5:8188）。` / `Check IP and port format (example: 192.168.1.5:8188).` | `修正后再试` / `Fix and retry` | `重试` / `Retry` | RETRY |
| TIMEOUT | `服务器没响应` / `Server didn't respond` | `连接超时。请确认 ComfyUI 已启动，并且手机和电脑在同一个 Wi-Fi 下。` / `Connection timed out. Make sure ComfyUI is running and your phone is on the same Wi-Fi as the computer.` | `检查 Wi-Fi 是否一致` / `Check Wi-Fi parity` | `重试` / `Retry` | RETRY |
| REFUSED | `连接被拒绝` / `Connection refused` | `服务器在线但拒绝了连接。可能 ComfyUI 没启动，或者端口被防火墙挡了。` / `The server is online but rejected the connection. ComfyUI may not be running, or the port is blocked.` | `检查 ComfyUI 是否启动 / 检查防火墙` / `Check that ComfyUI is running / check the firewall` | `重试` / `Retry` | RETRY |
| TLS_HANDSHAKE | `加密握手失败` / `TLS handshake failed` | `无法建立安全连接。试试不带 https，因为 LAN 模式默认是明文。` / `Couldn't establish a secure connection. Try without https — LAN mode uses plaintext by default.` | `去掉 https://` / `Remove https://` | `重试` / `Retry` | RETRY |
| NOT_COMFYUI | `这不像 ComfyUI` / `Doesn't look like ComfyUI` | `这个地址有响应，但不是 ComfyUI。请确认你输入了 ComfyUI 服务器的 IP，而不是别的服务。` / `Something is responding at that address, but it's not ComfyUI. Make sure you entered the ComfyUI server's IP, not another service.` | `换一个地址` / `Try a different address` | `重试` / `Retry` | RETRY |
| WRONG_PORT_404 | `ComfyUI 不在这个端口` / `ComfyUI isn't at this port` | `服务器在线，但这个端口上没找到 ComfyUI（/system_stats 返回 404）。换一个端口看看。` / `The server is reachable, but ComfyUI isn't at this port (/system_stats returned 404). Try a different port.` | `换个端口（默认 8188）` / `Try another port (default 8188)` | `重试` / `Retry` | RETRY |
| NO_ACTIVE_SERVER | `没有选中的服务器` / `No active server` | `还没有选好要连的服务器。从历史里挑一个，或在表单里输入 IP。` / `No server is selected. Pick one from history, or enter an IP again.` | `选服务器或重新输入` / `Pick a server or enter again` | `去选择` / `Choose` | **DISMISS** |
| UNKNOWN | `连接失败` / `Connection failed` | `不太确定问题出在哪儿。展开下面的"技术细节"看看原始信息。` / `Not sure what went wrong. Expand "Technical details" below for the raw message.` | (none — context is the action) | `重试` / `Retry` | RETRY |

**Action vs. CTA label invariant**: the CTA label and action must stay in sync. Reviewers should treat divergence as a bug (e.g. labelling a button `Choose` but routing it to `onRetry` would put a null-server retry into a loop). `ConnectErrorCopyTest` ratchets this lock — `if (primaryAction == DISMISS) primaryCtaEn != "Retry"`-style invariants live there.

**`NO_ACTIVE_SERVER` semantic** (added in PR #19, runner-side):
- Triggered by `ConnectionEffectRunner` when a side-effect intent that needs a server (e.g. `OpenWs`, `PollHistory`) is run while `ActiveServerHolder.current.value == null`. Runner emits `ConnectError.NO_ACTIVE_SERVER` and dispatches `ConnectAttempt(NO_ACTIVE_SERVER)` so the reducer lands on `Lost(NO_ACTIVE_SERVER)` — UI surfaces the error normally.
- **Zero IO** is initiated for that intent.
- The CTA `去选择 / Choose` wires to `onDismissError`, returning the user to the connect form/history (which is the screen background); it does NOT re-dispatch `Retry`, because retrying a null active server would only loop back into the same error.

**`Reset to imported workflow` button** is in the workflow view (T0.3 §6 safety valve), not in the connect flow. Don't add it here.

## 8. Acceptance contract (cross-ref T0.5)

DoD inherited from `docs/qa/T0.5-acceptance-and-regression-checklist.md`:
- **G-05** LAN recoverability — IP error / port unreachable / server offline / Wi-Fi switch / device sleep all surface a clear `ConnectError` from §5.
- Friendly_name + history persisted across cold restart (SQLDelight via T1.3).
- B branch (LAN_FLAKE) does **not** show alarmist banner.
- C branch (BACKGROUND_RESUMED) **must** show explicit banner.
- ETA does not jump after Reconnecting → Connected transition.
