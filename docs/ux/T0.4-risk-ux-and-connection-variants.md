## 1. 多指手势冲突矩阵（节点图编辑面）

### 1.1 输入 × 触发位置矩阵

| 输入 | 节点上 | 画布空白处 | 连线上 | 抽屉/侧栏 |
|---|---|---|---|---|
| **Tap** | 选中节点（高亮） | 取消选中 | （MVP 不响应） | 抽屉内交互 |
| **Long-press**（≥ 200ms） | 弹起底部抽屉/侧栏（编辑参数） | （MVP 无） | （MVP 无） | — |
| **Single-finger drag** | （MVP 不允许节点拖动 — 只读概览） | 平移画布 | （不响应） | 抽屉 handle 上：调整高度 |
| **Swipe up/down** | 让位给画布缩放/平移 | 平移画布 | — | 抽屉：上滑展开 / 下滑关闭 |
| **Two-finger pinch** | 缩放画布（统一行为） | 缩放画布 | 缩放画布 | — |
| **Two-finger drag** | 平移画布（双指优先） | 平移画布 | 平移画布 | — |

### 1.2 主要冲突 + 解决规则

| 冲突 | 解决规则 |
|---|---|
| 长按 vs 拖动起点 | 长按计时 200ms；时间内未达拖动阈值（位移 < 8dp）→ 长按；超阈值 → 取消长按计时，但 MVP 节点拖动禁用 → 直接吞掉 |
| 单指 swipe vs 双指 pinch/drag | 第二指落下立即取消单指序列，进入双指序列；单指未提交事件不发出 |
| 节点 tap vs 画布 tap | hit-test 节点 z-order 优先；命中节点 → 选中事件，不冒泡到画布 |
| 抽屉拖动 vs 画布拖动 | 抽屉 handle 区域命中优先；handle 外内容由抽屉子组件处理；抽屉边缘外的触控传给画布 |
| Tap 在已选中节点 | 第一次 Tap = 选中；第二次 Tap（同节点）= 弹起参数抽屉（手机省一次长按） |

### 1.3 特殊状态下的手势行为

| 状态 | 行为变化 |
|---|---|
| 选中"非白名单节点" | Tap 选中 → 显示 toast "此节点暂不支持移动端编辑，可在桌面 ComfyUI 中调整后重新导入"；Long-press 不弹抽屉 |
| 运行态（生图中） | 所有拖动/缩放禁用；仅 Tap 可查看节点状态详情；右上角始终保留"取消"CTA |
| 高级模式 toggle = ON（v1.5+） | 解锁节点拖动重定位、连线编辑（需 confirm + undo + pre-run validation 三件套） |

### 1.4 视觉/触觉反馈

- 触摸节点：100ms 内出现轻微 scale 0.98 + 阴影加深，给"被按住"反馈。
- 长按触发：触发瞬间 haptic light + 抽屉以 200ms ease-out 弹起。
- 选中节点：边框高亮（accent color，宽度 2dp）+ 浮起阴影。
- 双指 pinch 进入：取消单指 hover 状态，避免视觉冲突。
- 节点拖动（v1.5+）禁止区域：显示禁止 cursor + haptic warning。

## 2. 节点参数输入控件目录（基于 Alice T0.1 §3.5 的 10 种 ControlType + 3 表现层变体）

### 2.1 控件规约总表

| 控件 | 触控目标尺寸 | 视觉规约 | 错误态 | a11y |
|---|---|---|---|---|
| **Number** | 输入框 ≥ 44pt 高 | 数字键盘弹起；右侧显示单位（如有） | 红边框 + 行内错误文案 | role=textbox; label=参数名 |
| **Integer** | 输入框 ≥ 44pt | 整数键盘；可选 stepper 变体 | 同上 | role=textbox |
| **Integer-with-stepper**（变体） | 输入框 + 左右 +/- 按钮各 44×44pt | 步进按钮长按可加速；按钮在两端，输入框居中 | 同 Integer | role=stepper; aria-valuenow |
| **Seed special**（变体） | Integer + 🎲 随机按钮 + 🔒 锁定按钮（按钮 44×44pt） | 锁定状态下随机按钮 disabled；随机生成时数字滚动 0.3s 动画 | — | "随机种子"/"锁定种子"独立 label |
| **Slider** | 拇指 ≥ 24pt，触控区域扩展到 44pt | 拇指上方 popover 实时显示当前值 + 单位；预设标签栏（如 KSampler.steps 提供 Quick 20 / Default 30 / High 50） | — | role=slider; aria-valuemin/max/now |
| **Toggle** | 整体 ≥ 48×28pt | iOS/Android 平台默认样式 | — | role=switch |
| **SingleLineText** | 输入框 ≥ 44pt | 字符计数（如有 maxLen）；suffix 清空按钮 | 红边框 + 行内文案 | role=textbox |
| **MultilineText (Prompt)** | 输入框 ≥ 80pt 起，自动撑高至屏幕 50% | 字符计数；语法高亮（lora `<lora:foo:1.0>` / embedding `embedding:bar`）；trigger 弹层（输入 `<` → LoRA picker；`embedding:` → embedding picker）；右下角"展开为全屏编辑器"按钮 | — | role=textbox; multiline=true |
| **Dropdown** | 触发器 ≥ 44pt 高；下拉项 ≥ 44pt | 弹出底部 sheet（手机）或 popover（平板）；运行时拉取选项；空时显示 "正在加载…" / "无可用选项" | 选项加载失败显示重试按钮 + 错误归因 | role=combobox |
| **ModelPicker** | 触发器 ≥ 44pt（含模型缩略图） | 弹出全屏 picker：搜索栏 + 分组（最近 / 全部 / 收藏）；显示模型缩略图（如有）+ 文件大小；空时引导"在桌面 ComfyUI 安装模型" | 模型不存在（导入工作流引用了远端没有的模型）显示警告 + 选择替代模型按钮 | role=combobox |
| **ImagePicker** | 触发器 ≥ 60pt（含缩略图） | 触发底部 sheet：相册 / 相机 / 粘贴板 / 重置；选择后显示缩略图 + 替换 / 移除 | 上传失败显示重试 + 错误码 | role=button + 图片预览 alt |
| **Hidden** | 不显示在主面板 | 仅在 "详情" 折叠区里 read-only 列出 | — | — |

## 3. 运行态 / 等待态 / 中断态 / 失败态 UX × 3 连接分支

### 3.1 通用状态机（基于 Alice T0.4 thread msg c148571a 的 ConnectionState）

```
ConnectionState
├── Connected           ← A 默认（WS 健康）
├── Reconnecting(reason)  ← B/C 共用
│   ├── reason=LAN_FLAKE         (B：前台 WS 中断)
│   └── reason=BACKGROUND_RESUMED (C：app 切回前台)
└── Lost(errorClass)    ← 终态错误，显示恢复入口
```

UI 表现层根据 `reason` 决定渲染 banner 或静默心跳。

### 3.2 分支 A — 稳定 WebSocket（前台 + 健康 LAN）

UX 上限版：每步 ETA、当前节点高亮、节点级勾选。

**屏幕组件**：
- **顶部状态条**：左 = 工作流名 + "运行中"标签；中 = 步进条（current_step / total_steps）+ ETA 估算（基于过去 N 步均时）；右 = "取消"按钮（红色文本，destructive 样式）。
- **节点图区**（运行态）：当前节点高亮（accent 边框 + 节点 type 旁加旋转 spinner 图标）；已完成节点打 ✓；缓存命中节点打 ⚡（基于 `execution_cached` 事件）。
- **底部进度详情**（可折叠）：当前节点 type + 该节点的 (value/max) 进度（来自 `progress` 事件）+ 当前节点缩略图（如已开始产出）。
- **取消行为**：Tap "取消" → 显示 confirm sheet "确定取消？已生成的部分将丢失" → confirm 后：
  - **正在执行的 prompt** → 调 `POST /interrupt`（可带 `prompt_id`），UI 切到"取消中..." → 收到 `execution_interrupted` 后切到"已取消"页面。
  - **尚未开始（仍在 queue 里）的 prompt** → 调 `POST /queue` body `{"delete":[prompt_id]}`，立即移出队列；如需要可再走 `/history` 校验已不存在。
  - 提供"再次运行"和"返回"按钮。

**输入事件 → UI 行为**：
- `execution_start` → 切到运行态屏；初始化进度条 0/total。
- `execution_cached(nodes[])` → 这些节点全部立即打 ⚡。
- `executing(node)` → 当前节点高亮；如 node=null 表示空闲。
- `progress(value, max, node)` → 步进条更新；ETA 重新估算。
- `executed(node, output)` → 节点打 ✓；如 output 含图片三元组 `{filename, subfolder, type}`，缩略图通过 `/view` 异步拉取并显示在底部进度详情。
- `execution_success` → 切到结果页；展示完整产物。
- `execution_error` → 切到错误页；展示错误归类、当前节点、traceback（折叠展示）。
- `execution_interrupted` → 切到取消页。

### 3.3 分支 B — 前台 LAN 抖动（WS 中断 ≤ 5s）

**触发**：WS 连接断开、app 仍在前台。

**UX 表现**：
- 顶部状态条变化：步进条**保留最后已知进度**（不归零）；ETA 字样替换为"重连中…"+ 微弱呼吸动画（信任态原语）。
- 节点图区：保持当前选中节点的高亮，不变 disabled 视觉。
- 底部状态行追加一行小字："网络小波动，正在重连…"（不打断用户）。
- **5s 内重连成功** → 收到下一个 `progress_state` 事件 → UI 静默切回完整步进；不显示成功 toast（避免打扰）。
- **5s 未恢复** → 降级到 `/history/{prompt_id}` 轮询：每 3s 查询一次；UI 状态保持"重连中…"。
- **30s 仍未恢复** → 切到 `Lost(NetworkUnavailable)` → 显示底部 banner "已离线，请检查 Wi-Fi 后重试"，提供"重试"按钮。

### 3.4 分支 C — 后台挂起恢复（≥ 30s, WS 必断）

**触发**：app 切回前台时，发现有 in-flight prompt 但 WS 已断。

**UX 表现**：
- **回归 banner**（页面顶部）：橙色背景 banner "欢迎回来，正在检查你的生成…" + spinner，**显式可见**（不像 B 是静默）。
- 后台立即调用 `/history/{prompt_id}`：
  - **响应：已完成** → banner 切绿色 "✓ 你的生成已完成！" → 2s 后自动切到结果页（可点击"立即查看"跳过等待）。
  - **响应：仍在跑** → banner 切回 "继续生成中…"；尝试重连 WS；若 WS 重连成功，切回分支 A 行为；若失败，进入分支 B 的轮询。
  - **响应：失败/取消** → banner 切红 "生成失败" 或 "已取消" → 2s 后跳到错误/取消页。

**关键反约束**：
- 后台挂起期间**不展示假进度**。banner 文案明确说明"在后台运行，回来后更新"，不让用户期待实时进度。
- 不做前台 service / 静默推送保活（Alice T0.1 M1 标记为 v1 不做）。

### 3.5 信任态 UI 原语（B/C 共用）

| 元素 | 规约 |
|---|---|
| **心跳点** | 步进条上一个柔和的脉冲圆点（accent color），1.5s 周期，告诉用户"还在工作"。在分支 B 静默时使用。 |
| **"还活着"文案** | 永远以"正在…"或"…中"结尾的现在进行时；避免"已停止/失败"等 alarmist 词；分支 B "网络小波动，正在重连…"；分支 C "正在检查你的生成…"。 |
| **ETA 容差** | 重连后 ETA 不立即重新计算（避免突然跳动）；保留最近 3 步均时直到收到新事件；事件恢复后才平滑过渡到新 ETA。 |
| **取消按钮可用性** | 在 B/C 重连态下取消按钮**保留可见但 disabled**，提示"等待重连后可取消"；避免突然消失让用户困惑。Lost 态下重新启用，按"放弃此次任务"语义。 |
| **错误归类显示**（来自 Alice T0.1 M1） | Lost 态显示文案：格式错误 / 超时 / TLS / 非 ComfyUI / 错误端口 — 各自有不同的 actionable 建议。 |

### 3.6 验收点（与 @Lily T0.5 对齐）

- B/C 恢复后，若服务端 prompt 已完成，客户端必须通过 `/history` 补查 → 进入成功态，不能残留幽灵 `in_progress`。
- 后台挂起 ≥ 30s 后回到前台必须显示明确的 banner（C 入口可观察）。
- B 静默重连不应展示 alarmist banner（B 入口可观察）。
- ETA 在重连后不应大幅跳变（容差测试）。
