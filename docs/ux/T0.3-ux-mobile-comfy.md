## 1. 信息架构（Information Architecture）

### 主路径
```
启动
 └─ 首启动 → 找服务器（手动 IP + 历史 + 友好命名）
 └─ 后续启动 → 工作流列表（连最近用的服务器）
工作流列表
 └─ 导入入口（文件 / 粘贴 JSON / 系统 share-sheet）
 └─ 已导入工作流卡片
工作流视图
 ├─ 节点图区（默认只读概览，缩放/平移/选中）
 ├─ 参数编辑（手机底部抽屉，平板侧栏）
 └─ 运行 CTA → 运行态 → 结果 → 历史
历史
 ├─ 倒序列表 + 缩略图
 └─ 反查工作流（同工作流再生 / 微调 prompt 再生）
设置
 ├─ 服务器管理（编辑/删除/重命名）
 ├─ 自动保存到相册（**post-MVP** — 涉及 iOS/Android 相册权限、拒权恢复、隐私提示，T0.5 暂未列为 MVP gate）
 └─ 暗黑模式
```

## 2. 渲染层硬性要求清单（喂 T0.2 矩阵）

| # | 硬性要求 | 候选 A 纯 Compose Canvas | 候选 B Skiko 直绘 | 候选 C 平台分叉 |
|---|---|---|---|---|
| 1 | 同屏可见节点 ≥ 30（轻度用户场景上限） | ✅ Compose drawScope 命令式绘制，30 节点裕度足 | ✅ 性能更优 | ✅ 原生最优，但实现成本最高 |
| 2 | 必备手势：双指缩放 / 长按节点 / Tap 选中 / Swipe（**简化视图下不要求拖线连接、不要求多选框选**） | ✅ Compose `detectTransformGestures`/`detectTapGestures` 直支持 | ⚠️ 需自实现手势识别层 | ✅ 各平台原生手势 API |
| 3 | 节点参数控件**外置**（不嵌入 Canvas）：滑块 / 数字 / 多行 prompt+autocomplete / 单选 / 下拉 / 图片选择器 | ✅（控件由抽屉/侧栏承担，Canvas 只画节点本体） | ✅（同） | ✅（同） |
| 4 | Workflow 节点绘制：矩形 / 标签 / IO 端口 / 贝塞尔连线 / 状态高亮 | ✅ drawScope 全套 | ⚠️ 需自实现文本布局 | ✅ 原生最佳 |
| 5 | 动效预算：节点拖动 60fps / 抽屉过渡 ≤200ms / 进度动画 | ⚠️ 30 节点 + 复杂连线 + 60fps 接近 Canvas 极限，需**视口虚拟化** + **连线缓存** | ✅ 性能裕度更大 | ✅ 原生最佳 |
| 6 | 平板分屏布局（左节点图 / 右参数+预览） | ✅ Compose Multiplatform 响应式承担，与渲染层无关 | ✅ 同 | ✅ 同 |
| 7 | 暗黑模式 + 主题切换 | ✅ MaterialTheme 原生 | ⚠️ 需自实现主题 | ✅ 原生主题 API |
| 8 | 无障碍（TalkBack/VoiceOver） | ✅ Compose 语义节点 | ⚠️ Skiko 自绘需手工挂 a11y 节点 | ✅ 原生组件天然 a11y |
| 9 | i18n / RTL | ✅ Compose 原生 | ⚠️ 需自实现 | ✅ 原生 |

### 推荐方向

- **默认 A（纯 Compose Canvas）**：满足轻度用户场景（≤30 节点 + 简化交互），动效紧但可达；a11y / 暗黑 / i18n 顺路获得；KMP 工程最简。
- **B（Skiko 直绘）作为升级路径**：未来若复杂度上升或需要更高节点密度。
- **C（平台分叉）作为终极后备**：MVP 不启用，工程与维护成本过高。

### 对 UX 的反向约束（cost hint 邀请 @Alice）

如候选 A 在 "30 节点 + 60fps + 视口虚拟化 + 连线缓存" 上撑不住，请反馈，我可对 UX 上限做以下任一调整：
- (a) 同屏节点上限压到 ≤20（对轻度用户仍可用）
- (b) 节点拖动从 60fps 降到 30fps（节点图编辑场景可接受）
- (c) 连线渲染降级（贝塞尔 → 直角折线，减少计算）

## 6. 已收口的决议（Phase 0）

- **Q1（unknown 节点拓扑可改？）**：三方一致 (Lily 评审 + Alice 架构 + 我 UX) **不允许**；安全阀 = "重置到导入时" 按钮 + 误触 toast；v1.5+ 解锁需"确认 + 撤销 + 运行前校验"三件套。@nothing 不否决视为采纳（Priestess msg c7dd63ae）。
- **渲染层**：Candidate A（Compose Canvas），Alice 的 4 步降级方案（视口虚拟化 / 静态层缓存 / 拖动期连线降级 / Compose 重组优化）。
- **ControlType**：10 种 base + 3 个 UI 表现层变体（不进 enum）。
- **Whitelist canonical**：用真实 classType 作主键，displayName 归描述符 entry。
- **PNG roundtrip**：`extra_data.extra_pnginfo.workflow` 嵌入 **current edited UI snapshot**（不是 original imported），由 @Lily 在 T0.6 review 中抓出来后修正。
