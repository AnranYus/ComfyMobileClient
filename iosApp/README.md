# `iosApp/` — iOS shell

Compose Multiplatform iOS app. Phase 1.0 will add the `xcodeproj` and Swift entry point.

## Planned structure (Compose Multiplatform pattern)

```
iosApp/
├── iosApp.xcodeproj/         # Xcode project — generate via Xcode + KMP plugin
└── iosApp/
    ├── iOSApp.swift          # App entry; wraps ComposeRoot
    ├── ContentView.swift     # SwiftUI host that calls Main.kt's MainViewController
    ├── Info.plist
    └── Assets.xcassets/      # AppIcon, accent color
```

## Bridging convention

`:shared` exports a `framework` named `shared.framework`. The Swift side calls:

```swift
import shared
struct ContentView: View {
  var body: some View {
    ComposeView().ignoresSafeArea(.keyboard)
  }
}
struct ComposeView: UIViewControllerRepresentable {
  func makeUIViewController(context: Context) -> UIViewController {
    Main_iosKt.MainViewController()
  }
  func updateUIViewController(_ vc: UIViewController, context: Context) {}
}
```

Where `Main_iosKt.MainViewController()` is exposed from `:shared:src/iosMain/kotlin/com/comfymobile/Main.ios.kt`.

## Why no `xcodeproj` yet?

The Xcode project file is binary-ish XML and is best generated by Xcode (or Compose Multiplatform's `kotlin xcode` integration). Phase 1.0 will run that generation step on macOS and commit the result.

## Permissions / Info.plist (planned)

- `NSLocalNetworkUsageDescription` — required on iOS 14+ for LAN connections to a user-entered IP.
- `NSBonjourServices` — leave empty in v1 (no mDNS); placeholder for future iteration.
- No camera/microphone usage in v1.
