## Xcode Build

Prerequisites:

- macOS with Xcode installed
- JDK 17 available to Gradle

Open `iosApp/iosApp.xcworkspace`, select the `iosApp` scheme, and run on an iOS simulator or a connected device.

The Xcode target has a `Build shared KMP framework` script phase:

```bash
cd "$SRCROOT/.."
./gradlew :shared:embedAndSignAppleFrameworkForXcode --stacktrace
```

That phase builds the correct `shared.framework` variant for the selected SDK and configuration before Swift sources compile.

## Command-Line Smoke Build

Simulator build:

```bash
xcodebuild \
  -workspace iosApp/iosApp.xcworkspace \
  -scheme iosApp \
  -configuration Debug \
  -destination 'platform=iOS Simulator,name=iPhone 16' \
  build
```

Unsigned device archive, matching the PR CI workflow:

```bash
xcodebuild \
  -workspace iosApp/iosApp.xcworkspace \
  -scheme iosApp \
  -configuration Release \
  -sdk iphoneos \
  -destination 'generic/platform=iOS' \
  -archivePath "$PWD/build/ios/ComfyMobileClient.xcarchive" \
  CODE_SIGNING_ALLOWED=NO \
  CODE_SIGNING_REQUIRED=NO \
  CODE_SIGN_IDENTITY="" \
  archive
```

The unsigned archive proves the project links and archives on `macos-latest`; it is not installable on a physical device.

## Physical Device Testing

For local device testing, open the workspace in Xcode and set a development team for the `iosApp` target. Xcode can then sign a Debug build directly to a connected device.

For ad-hoc sideload distribution, add an Apple Distribution certificate and matching provisioning profile to CI and switch the archive/export step to signed mode. Until those secrets exist, CI intentionally uploads only the unsigned `.xcarchive.zip` artifact.

## Local Network Permission

`Info.plist` includes `NSLocalNetworkUsageDescription` because ComfyMobile connects to a user-entered ComfyUI server on the LAN. `NSBonjourServices` is present but empty because v1 does not do mDNS discovery.
